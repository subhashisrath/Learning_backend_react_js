# Learn While Building Protocol (ParkingAutomation)

Use this with the 7-day plan so you understand every line.

## Session Rule

For every file you build, do these 5 phases:

1. Preview (5-10 min)
- Read file once.
- Mark unknown words/syntax.
- Write what you think file does in 2 lines.

2. Rebuild (20-60 min)
- Type code manually in your learning project.
- Do not copy large blocks.

3. Explain (10-20 min)
For each logical block, answer:
- What does it do?
- Why needed?
- What if removed?

4. Break/Fix (10-15 min)
- Intentionally break 1 line.
- Observe error.
- Fix it and note the reason.

5. Improve (10-20 min)
- Add one tiny change by yourself.
Examples: rename labels, add one field, add a filter, add loading text.

## Concept Tags (track as you learn)

- JS: arrays, objects, map/filter/reduce, spread
- React: components, props, state, effect, refs
- Router: routes, navigate, guards, outlet
- UI: conditional render, list render, forms, events
- App: auth, role permissions, API layer, charts

## Understanding Score (self-check)

Give each file a score before moving on:
- 0 = copied, no idea
- 1 = partial idea
- 2 = can explain most lines
- 3 = can rewrite from memory

Only move to next major file if score is >=2.
